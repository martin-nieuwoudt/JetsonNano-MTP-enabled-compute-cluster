Resize linux image.md            :: Critical Warning Before You StartBackup your file: Copy your .img file to a safe location before continuing.Do not guess sizes: The file container must always remain slightly larger than the filesystem inside it.Tool requirement: Because Windows cannot read or modify Linux filesystems (like ext4), you will need a Linux environment to safely resize the internal data. If you do not have WSL installed, you can use a free tool like OSFMount for Windows or boot into a live GParted ISO via a virtual machine.Step 1: Shrink the Linux FilesystemYou must safely shrink the internal file system first. This tells the data to compress into the front section of the disk space. This step requires a Linux environment (like WSL or a Linux Virtual Machine).Connect your .img file to a Linux loop device:sudo losetup -fP /mnt/c/path/to/your/image.imgFind the loop device path (e.g., /dev/loop0) by running:losetup -aForce a file system check to ensure there are no errors:sudo e2fsck -f /dev/loop0p1Shrink the filesystem to your target size (for example, shrinking it down to 15 Gigabytes):sudo resize2fs /dev/loop0p1 15GStep 2: Shrink the Partition BoundaryNow that the data is compressed into the first 15GB of space, you must move the partition wall to match.Open the partition tool:sudo parted /dev/loop0Type print to see your current partitions and note the partition number (usually 1).Shrink the partition boundary to match your exact filesystem size:resizepart 1 15GBType quit to exit.Cleanly disconnect the image file from Linux:sudo losetup -d /dev/loop0Step 3: Truncate the Physical .img File ContainerNow that the data and partition walls are safely rolled back, you can safely cut off the empty trailing space using Windows.Open your Windows Command Prompt as an Administrator.If you use QEMU for Windows, use the resize command with a negative value to shrink the outer container to match your new target size:qemu-img.exe resize C:\path\to\your\image.img 16G(Note: Always make the outer container slightly larger, e.g., 16GB, than the internal partition to avoid cutting off critical sector data).Alternatively, you can use the free windows tool VBoxManage if you use VirtualBox:VBoxManage modifymedium disk C:\path\to\your\image.img --resize 16384 (Size in Megabytes)
